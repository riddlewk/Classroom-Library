# Documentation

All project requirements and planning documents are stored in the `docs/` directory.

## Core Documents

- **01-product-vision.md** — High‑level purpose, goals, and user value of the Classroom Library app.
- **02-functional-requirements.md** — Detailed feature list: ISBN lookup, auto‑tagging, CRUD, auth, etc.
- **03-nonfunctional-requirements.md** — Performance, security, platform, and maintainability constraints.
- **04-architecture.md** — System architecture overview including client, Supabase, and external APIs.
- **05-data-model.md** — Supabase/Postgres schema: tables, fields, relationships, and RLS notes.
- **06-ux-ui-spec.md** *(optional placeholder)* — Wireframes, UI flow, and interaction notes.
- **07-isbn-integration.md** — How ISBN lookup works using Open Library + Google Books.
- **09-project-plan.md** — Phase‑based development roadmap for building the application.
